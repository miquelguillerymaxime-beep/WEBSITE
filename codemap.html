How · it works

Parse, graph, query.

No model runs while your code is indexed. The structure comes straight from the syntax tree, so the map is always fresh and free to update.

01

Parse the tree

Tree-sitter reads every file into an abstract syntax tree. Zero LLM calls, so it stays fresh on every edit.

02

Build the graph

Symbols become nodes, calls and imports and type references become typed edges. 1,500 files in 1.5 seconds.

03

Answer the query

Your agent asks in plain terms. The graph returns the real answer with its edge path, in 13 milliseconds.
