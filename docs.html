Get your agent reading your repo.

Three commands and a first query. No model runs while your code is indexed, so it stays fresh on every edit.

Quickstart

1. Install the CLI. One command, no dependencies to configure.

# install
curl -fsSL memtrace.io/install.sh | sh

2. Connect your repo. Sign in with GitHub or Google, then point Memtrace at a repository.

memtrace connect ./my-repo

3. Index. The graph builds in about 1.5 seconds per 1,500 files, locally.

memtrace index
# 1,504 symbols · 13.4ms · ready

Your first query

Ask the question a search index cannot answer: the blast radius of a change.

memtrace impact rotateToken
# called by 7 · affects 12 · 1 risk in payments/charge.rs

This is the moment most people understand what Memtrace does. Try it on a function you actually plan to change.

Code map

Every function, file and dependency, mapped into one graph. Your agent queries the real call graph, not a text match. See the Code map page for the full picture.

  • find_symbol, locate the real definition
  • get_impact, the blast radius of a change
  • get_symbol_context, callers, callees and history

MemDB

Memory with time built in. Every record knows when it was true, so you can ask a query as-of any past moment and get the value that held then. Written in Rust, runs on a laptop. More on the MemDB page.

Fleet

When you run more than one agent, they share the graph instead of fighting each other. Agents announce intent before editing, and overlaps auto-resolve. More on the Fleet page.

MCP setup

Memtrace speaks the Model Context Protocol, so your agent reads the graph as a native tool. Add the server once.

claude mcp add memtrace -- memtrace serve

Your agent now has find_symbol, get_impact and the rest available on every turn.
